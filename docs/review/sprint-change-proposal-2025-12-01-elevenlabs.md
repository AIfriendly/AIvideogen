## Section 1: Issue Summary

**Problem Statement:**
The current TTS system only supports Kokoro (local, FOSS) for voiceover generation. Users who want premium, cloud-based voice quality have no alternative option within the application.

**Discovery Context:**
User request to add ElevenLabs API as an alternative TTS provider alongside Kokoro, with separate voice catalogs and usage tracking.

**Evidence:**
- Current system locked to single TTS provider (Kokoro)
- ElevenLabs offers premium voice quality via API
- Aligns with "FOSS-first, cloud-enhanced" philosophy (Kokoro = FOSS, ElevenLabs = cloud option)
- Follows existing provider pattern (Ollama/Gemini for LLM)
